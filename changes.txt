[2026-07-15 15:47 IST] Wallet integrity: money is never manufactured
  - Fixed the reported bug: with ₹40,000 in the wallet and a ₹35,000 booking,
    triggering the disruptor rebooked a ₹45,000 replacement and confirmed it.
    Three silent money-creation paths were removed:
    1. agents/agent5_disruption.py auto-topped-up the wallet to force an
       unaffordable rebooking through. Now: the cancellation + refund stand,
       the rebooking is blocked, and the response says exactly how much to
       top up (status cancelled_only, reconciliation action top_up_required).
    2. The same flow then ran wallet.reconcile(), which moved the price
       difference AGAIN on top of the real refunds and rebooking debit — a
       double charge. The reconciliation is now an informational summary.
    3. orchestration/orchestrator.py confirm_and_book auto-credited any
       shortfall (need + 50). Now the confirm fails with a clear top-up
       message; explicit top-ups still work via topup_if_needed / wallet UI.
  - Crash-window compensation in agents/agent3_booking.py: an in_progress
    booking row is persisted before any wallet debit, and a retry after a
    crash refunds the orphaned debits (ledger-derived) before booking afresh.
  - New tests/test_wallet_integrity.py covers all four behaviors.

[2026-07-15 16:10 IST] Chat start times respect the user's timezone
  - Found during live verification: telling the chat "9:00 AM" produced an
    appointment at 09:00 UTC (14:30 IST) because _resolved_start stamped the
    server's UTC on the user's wall-clock time. It now resolves "today"/
    "tomorrow" and stamps the result in the client's reported timezone.
  - Added the tzdata package to requirements.txt: the environment's zoneinfo
    was missing legacy aliases like Asia/Calcutta (what browsers actually
    send), which silently fell back to UTC; Windows has no system tz data at
    all. New regression test in tests/test_chat_agent.py.

[2026-07-15 16:10 IST] Dev launch config fixed for Linux
  - .claude/launch.json dmos-api pointed at .venv/Scripts/python.exe (a

[2026-07-15 15:47 IST] Fine-tuned model wired as an optional LLM provider
  - LLM_PROVIDER=local_lora loads the LoRA adapter in-process via
    finetuning/inference.py (LORA_ADAPTER_PATH/LORA_BASE_MODEL); if the ML
    stack is missing the client disables itself and the deterministic
    controller takes over. LLM_PROVIDER=local targets a local OpenAI-
    compatible server (vLLM/llama.cpp/Ollama) with no API key required.
    Documented in .env.example and finetuning/README.md.

[2026-07-15 12:48 IST] CORS allows any localhost port in dev
  - api/main.py adds allow_origin_regex for http://localhost:<any port> and
    http://127.0.0.1:<any port>. The explicit origin list only covered ports
    3000/8501, which silently blocked local tooling (previews, ad-hoc dev
    servers) on other ports. Remote sites cannot forge a localhost Origin,
    so this remains dev-safe; real origins must be listed explicitly before
    any internet-facing deployment.
